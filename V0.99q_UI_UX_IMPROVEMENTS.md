### 3. Independent Time Refresh Mechanism

**Problem:** With long price update intervals (e.g., 5-10 minutes), the clock wouldn't update for the entire interval, making CryptoBar unsuitable as a desk clock.

**Solution:**
- Added independent minute-aligned time refresh
- Clock updates every minute via partial refresh
- Time refreshes **do NOT count** toward the 20-refresh partial limit
- Price updates continue on their own schedule

**Implementation Details:**

#### New Global Variables (`app_state.h/cpp`):
```cpp
time_t g_nextTimeRefreshUtc;   // Next time-only refresh timestamp
bool   g_timeRefreshEnabled;    // Enable/disable (default: true)
```

#### New UI Function (`ui.h/cpp`):
```cpp
void drawMainScreenTimeOnly(bool fullRefresh = false);
```
- Partial window: 50-pixel height at top of white panel
- Updates only date/time area
- Does not redraw price, chart, or symbol panel

#### Main Loop Logic (`main.cpp`):
1. **Time Refresh Check** (line 813-837):
   - Runs every loop iteration
   - Checks if current time ‚â• next scheduled time refresh
   - Aligns to minute boundaries: `(nowUtc / 60 + 1) * 60`
   - Calls `drawMainScreenTimeOnly(false)` for partial refresh
   - Does NOT increment `g_partialRefreshCount`

2. **Reset After Price Update** (line 999-1003):
   - When price updates, time also updates
   - Resets `g_nextTimeRefreshUtc` to next minute boundary
   - Prevents duplicate time refresh immediately after price update

**Example Timeline** (5-minute price refresh, Full refresh mode):
```
10:00 ‚Üí Price refresh (FULL) + reset time scheduler
10:01 ‚Üí Time partial refresh (not counted)
10:02 ‚Üí Time partial refresh (not counted)
10:03 ‚Üí Time partial refresh (not counted)
10:04 ‚Üí Time partial refresh (not counted)
10:05 ‚Üí Price refresh (FULL) + reset time scheduler
10:06 ‚Üí Time partial refresh (not counted)
...
```

## üîß Post-Release Hotfixes (User Feedback)

### Issue 1: Time-Only Refresh Causes Price Display Artifacts

**Problem:** The initial implementation used a 50-pixel partial window for time refresh, which caused the price area to display incorrectly (top portion erased).

**Root Cause:** GxEPD2's partial refresh mechanism affects surrounding pixels when using a small partial window.

**Solution (Applied):**
- Changed `drawMainScreenTimeOnly()` to use **full-area partial refresh**
- Sets partial window to entire white panel: `display.setPartialWindow(SYMBOL_PANEL_WIDTH, 0, width, height)`
- Redraws all content (time, price, chart) but only time data changes
- GxEPD2 compares old vs new framebuffer and only refreshes changed pixels (time area)
- Prevents artifacts around price/chart areas

**Files Modified:**
- `src/ui.cpp` (lines 955-983): Updated `drawMainScreenTimeOnly()` implementation

---

### Issue 2: Time Refresh Not Coordinated with Price Refresh

**Problem:** Time refresh executed independently of price refresh, causing duplicate refreshes at the same timestamp.

**Example Issue:**
```
10:05:02 ‚Üí Time refresh (partial)
10:05:05 ‚Üí Price refresh (full/partial)  ‚Üê Wasted the time refresh
```

**Root Cause:** Time refresh logic executed before checking `doUpdate` flag.

**Solution (Applied):**
- Moved time refresh logic to execute **after** `doUpdate` check
- Added `!doUpdate` condition: only refresh time if price refresh is NOT scheduled
- When `doUpdate=true`, time refresh is skipped (price refresh includes time update)
- Price refresh resets time refresh schedule to next minute boundary

**Execution Priority:**
1. Check if price refresh needed (`doUpdate`)
2. If `!doUpdate`, check if time refresh needed
3. Execute price refresh (if needed) and reset time schedule
4. Time and price refreshes never execute at the same timestamp

**Corrected Timeline (5-minute price refresh):**
```
10:00 ‚Üí doUpdate=true  ‚Üí Price refresh (includes time) ‚úÖ Skip time refresh
10:01 ‚Üí doUpdate=false ‚Üí Time partial refresh
10:02 ‚Üí doUpdate=false ‚Üí Time partial refresh
10:03 ‚Üí doUpdate=false ‚Üí Time partial refresh
10:04 ‚Üí doUpdate=false ‚Üí Time partial refresh
10:05 ‚Üí doUpdate=true  ‚Üí Price refresh (includes time) ‚úÖ Skip time refresh
10:06 ‚Üí doUpdate=false ‚Üí Time partial refresh
...
```

**Files Modified:**
- `src/main.cpp` (lines 813-837 removed, 883-909 added): Relocated and updated time refresh logic

### Issue 3: Price Refresh Executes 6 Seconds Before Minute Boundary

**Problem:** Price refresh executed at 6 seconds before the minute boundary (e.g., 10:04:54 instead of 10:05:00), causing misalignment with time refresh.

**Example Issue:**
```
10:04:54 ‚Üí Price refresh (6 seconds early) ‚ùå
10:05:00 ‚Üí Time refresh (correct timing)
```

**Root Cause:** V0.99o MAC jitter logic incorrectly subtracted jitter from `g_nextUpdateUtc` (screen refresh time) instead of only applying it to prefetch timing.

**Original Logic (Incorrect):**
```cpp
// app_scheduler.cpp line 40-46 (before fix)
time_t nextDisplayUtc = alignNextTickUtc(nowUtc, sec);  // 10:05:00
uint32_t totalLead = MIN_API_LEAD_SEC + g_fetchJitterSec;  // 6 + jitter
g_nextUpdateUtc = nextDisplayUtc - totalLead;  // 10:05:00 - 6 = 10:04:54 ‚ùå
```

This caused `doUpdate` to trigger at 10:04:54 instead of 10:05:00.

**Corrected Logic:**
```cpp
// app_scheduler.cpp line 38-41 (after fix)
// g_nextUpdateUtc is the screen refresh time (epoch-aligned)
// All devices refresh screens at the same minute boundary
// Prefetch logic (in main.cpp) handles API request jitter separately
g_nextUpdateUtc = alignNextTickUtc(nowUtc, sec);  // 10:05:00 ‚úÖ
```

**Solution (Applied):**
- Removed jitter calculation from `tickSchedulerReset()`
- `g_nextUpdateUtc` now always aligns to minute boundaries
- Prefetch mechanism (already in main.cpp line 824) handles jitter independently:
  ```cpp
  time_t prefetchAtUtc = g_nextUpdateUtc - PREFETCH_FIXED_LEAD_SEC;
  // Correctly calculates: 10:05:00 - 6 = 10:04:54 for API prefetch
  ```

**Execution Timeline (5-minute price refresh):**
```
10:00:00 ‚Üí doUpdate=true  ‚Üí Price refresh ‚úÖ
10:01:00 ‚Üí doUpdate=false ‚Üí Time refresh
10:02:00 ‚Üí doUpdate=false ‚Üí Time refresh
10:03:00 ‚Üí doUpdate=false ‚Üí Time refresh
10:04:00 ‚Üí doUpdate=false ‚Üí Time refresh
10:04:54 ‚Üí Prefetch API call (6s early, with MAC jitter)
10:05:00 ‚Üí doUpdate=true  ‚Üí Price refresh ‚úÖ (uses prefetched data)
```

**Result:**
- ‚úÖ Price refresh executes at minute boundary (10:00, 10:05, 10:10...)
- ‚úÖ Time refresh executes at minute boundary (10:01, 10:02, 10:03...)
- ‚úÖ API prefetch executes 6 seconds early with MAC jitter (distributes load)
- ‚úÖ Multiple devices refresh screens simultaneously (synchronized display)
- ‚úÖ API requests are staggered (6-16 seconds before display, prevents server overload)

**Files Modified:**
- `src/app_scheduler.cpp` (lines 38-50): Removed incorrect jitter from `g_nextUpdateUtc`
